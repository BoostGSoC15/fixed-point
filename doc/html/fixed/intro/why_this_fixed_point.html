Why use Boost.Fixed-point?

A number of fixed-point libraries have been written; why would one choose this library?

  • Boost license, free for all uses, including commercial.
  • Compatible with C++ standard including std::numeric_limits and C math libraries.
  • Range and resolution are chosen using C++ templates.
  • Rounding and overflow may be controlled using C++ templates.
  • The type having the smallest underlying power-of-2 bits (int8_t, int16_t, int32_t, int64_t, etc.) will be chosen automatically for the internal representation.
  • Extends to use underlying fundamental types automatically.
  • Extends automatically to use Boost.Multiprecision when the resolution and/or range specified are too big to fit into the largest of the fundamental type. This allows fixed-point type with much higher resolution and/or range than any other library. This is a unique feature.
  • Convenient header-only implementation (even when extended to Boost.Multiprecision).
  • Transparent testing.
