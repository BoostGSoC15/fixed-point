Discusses how using only a 16-bit floating-point type as a storage type can save much memory when there are very many values to store. Using less memory can also reduce cache misses and can reduce disk I/O. The type is always converted to a fundamental floating-point type for any computation; instructions vcvtph2ps and vcvtps2ph are provided to convert to and from 32-bit float efficiently.
