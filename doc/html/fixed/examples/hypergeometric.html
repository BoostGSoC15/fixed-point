Example of computing hypergeometric_1f1 function

This example shows computation of the hypergeometric 1f1 function using a high precision (and high resolution) fixed_point type. The rational approximation, similar to that of Y.L. Luke, "Algorithms for the Computation of Mathematical Functions", Academic Press, New York, 1977, is used to compute hypergeometric_1f1(2/3, 4/3, -3/4). A 4-term recurrence relation is computed with 340 iterations at full precision of approximately 963 decimal digits. The result also has the same approximate precision of 963 decimal digits.

Wolfram Alpha Hypergeometric1f1[2/3,4/3,-3/4] estimates 0.7081686536671296176070489154441892975263864254728...

See also hypergeometric function.

Using fixed_point, this example takes approximately 0.16 sec on a modern computer and compiler system. For comparison, the same calculation performed with Boost.Multiprecision's binary floating-point backend only requires approximately 0.12 sec with the same computer and compiler system. In this particular case, there is no significant performance improvement for fixed-point. In fact, there is a slight slowdown for fixed-point.

Even though a performance improvement was not obtained in this example, the calculation does exercise the Boost.Fixed-point Library with complex algorithms at high precision.

The instantiated fixed-point class in this example is: boost::fixed_point::negatable<122, -3200>> and the instantiated multiprecision class is: boost::multiprecision::number<class boost::multiprecision::backends::cpp_bin_float<963>>.

Full code for the fixed-point calculation is at fixed_point_hypergeometric_1f1_fixed.cpp.

Full code for the floating-point calculation is at fixed_point_hypergeometric_1f1_float.cpp.
