Frequently Asked Questions FAQ

  1. What compiler do I need?
    Many C++11 features are used in the Boost.Fixed-point Library and the compiler must have a high level of C++11 awareness to successfully translate the source code. Visual Studio 2015 (VC 14.0) or later works well, but many applications will also work with Visual Studio 2013 (VC 12.0). GCC compilers versions such as 5.x or later or Clang with similar C++11 adherence also work well.
  2. When should I consider using a fixed-point type?
    Fixed-point allows you to provide a lower or higher range and/or a lower or higher precision.
  3. Will a fixed-point type use less memory?
    If you base the fixed-point type on a small underlying integer type, then it will be more efficient.
  4. Will a fixed-point type run faster than floating-point type?
    If there is no hardware floating-point, then is very likely to run much faster. If you require a very high range or precision that would require Boost.Multiprecision or similar high or arbitrary precision type, then for the same precision or range, it will probably be quicker, possibly much quicker.
  5. How can I avoid using iostream on a bare-metal microcontroller without any streaming peripherals?
    #define BOOST_FIXED_POINT_DISABLE_IOSTREAM.
  6. Can I use multiprecision without iostream on a bare-metal microcontroller without any peripherals?
    This is not possible because the multiprecision software was written assuming iostream is available. The work to isolate use of iostream using a preprocessor macro is judged too great to be worthwhile just for this unusual requirement.
