Preprocessor Options to enable/disable some features

Some macros are available (or are becoming available) to enable/disable certain library features that may improve the usefulness of fixed_point in certain application domains.

  • BOOST_FIXED_POINT_DISABLE_IOSTREAM All I/O streaming is disabled, as is the inclusion of associated standard library headers. This option eliminates all I/O stream overhead, in particular for bare-metal microcontroller projects. Disabling I/O streaming requires simultaneous disabling of multiprecision, as shown below with BOOST_FIXED_POINT_DISABLE_MULTIPRECISION. (Implemented).
  • BOOST_FIXED_POINT_DISABLE_MULTIPRECISION This option is defined to disable the use of Boost.Multiprecision for back-ends of the fixed-point classes. Multiprecision fixed-point numbers are not available if this option is set. (Implemented).
  • BOOST_FIXED_POINT_DISABLE_WIDE_INTEGER_MATH This option is defined to avoid using the unsigned_large_type in the implementations of multiplication and division operations. This option is intended for systems with limited integer widths such as bare-metal microcontrollers. When used in combination with BOOST_FIXED_POINT_DISABLE_MULTIPRECISION, this option is intended to provide fixed-point representations with up to 64-bits (if 64-bit integer types are available) without requiring any multiprecision. (Not yet implemented).
  • BOOST_FIXED_POINT_DISABLE_IOSTREAM is defined to disable all I/O streaming and the inclusion of associated standard library headers.
    This is intended to eliminate all I/O stream overhead, in particular for bare-metal microcontroller projects. (Implemented). For an example, see fixed_point_no_io.cpp.
  • BOOST_FIXED_POINT_DISABLE_CPP11 This option enables support for a back-port to C++03. This option eliminates the use of all native C++11 language elements. This might send the wrong message about language technology, but could increase the range of potential target compilers, especially for embedded systems. (Not yet implemented).
  • BOOST_FIXED_POINT_ENABLE_GMP_BACKENDS When this option is defined, fixed-point uses multiple-precision backends (when required) based on GNU Multiple Precision Arithmetic Library (GMP) instead of Boost.Multiprecision's self-written binary floating-point backend. See GMP float types section in Boost.Multiprecision. GNU Multiple Precision Arithmetic Library (GMP) backends use highly optimized assembler and advanced multiplication algorithms that are significantly faster for high digit counts in excess of a few hundred binary digits. (Implemented but not fully tested).
