Infinity and NotANumber (NaN)

Unlike floating-point types, the fixed-point types (may) occupy the entire underlying integer type, and no bit patterns are reserved for NaN or infinity. The bool std::numeric_limits<T>::has_infinity and bool std::numeric_limits<T>::has_quiet_NaN both return false for all fixed-point types.

Lack of an encoding for infinity or NaN means that overflow or underflow must be signalled differently.

A template parameter typename OverflowMode is provided to permit control what happens on overflow or underflow. In namespace overflow some struct like modulus, exception etc are provided.
