Testing

Testing is focussed on typical use cases, stress cases, and boundary case, edge case and corner case cases. Test coverage has not been used during testing, but may optionally be included in future validation.

A strong effort has also been made to test all arithmetic functions, mixed-precision operations, transcendental functions and IO streaming.

A variety of other non-trivial calculations has also been included in the test suite. These include calculus operations, mathematical constants, special functions, root-finding and more. Some of these tests have been specifically desinged to use Boost.Math in order to partially verify interoperability with Boost.Math.

Tests have been created for all key precision ranges. Both supported rounding modes have been tested. Certain calculations in the tests have been extended to very few digits of precision and also to thousands of digits of precision in order to stress the limits of the library. Some tests have also been run on 8-bit and 32-bit deeply embedded bare-metal microcontrollers, strengthening verification of the library's portability.

Extensive round-trip testing has been performed over a wide range of precision. This involves constructing a fixed_point type from decimal digits strings like '0.001, 0.002 ...' This fixed-point representation is output to a stringstream and then read back into another fixed-point variable, and the two checked for equality,

These kinds of tests could be carried out for all possible values of small bit-count fixed-point types, but this would take unfeasible test times for more than about 32 bits, so roughly random values covering the entire range of possible binary patterns are used for test values instead.

Note

History - Random testing was used to track bugs in a popular compiler's stream input from decimal digit strings. A fault was originally discovered by a user of serialization when he found (after much debugging) a single value of de-serialized data that was 1-bit different from that originally written. Random value testing then revealed that a small range of values could not be round-tripped in this way, and a third of the values in this small range were 1-bit different. The C++17 draft standard does not (yet) require input from decimal digit strings to the nearest representable value, but while other examples were also found in all compilers, most are now achieve correct conversion for all values.
