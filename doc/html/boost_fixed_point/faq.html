Frequently Asked Questions FAQ

  1. When should I consider using a fixed-point type?
    Fixed-point allows you to provide a lower or higher range and/or a lower or higher precision.
  2. Will a fixed-point type use less memory?
    If you base the fixed-point type a small underlying integer type, then it will be more efficient.
  3. Will a fixed-point type run faster than floating-point type?
    If there is no hardware floating-point, then is very likely to run much faster. If you require a very high range or precision that would require Boost.Multiprecision or similar high or arbitrary precision type, then for the same precision or range, it will probably be quicker, possibly much quicker.

Why do I get compiler errors when passing a number to a template function?

Most likely you are actually passing an expression template type to the function and template-argument-deduction deduces the "wrong" type. Try casting the arguments involving expressions to the actual number type, or as a last resort turning off expression template support in the number type you are using.
